# Clox VM: High-Performance C Bytecode Virtual Machine

A lightweight, high-performance stack-based bytecode virtual machine and single-pass compiler for the Lox programming language, implemented in pure C99. 

This project is a complete implementation of the virtual machine and compiler design, featuring custom memory management, an optimized hash table, lexical closures, object-oriented programming constructs, and a precise tracing garbage collector.

---

## 🚀 Key Features

* **Stack-Based Virtual Machine:** Direct execution of custom bytecode instructions with a highly optimized instruction decoding loop.
* **Single-Pass Pratt Parser:** Direct compilation from source text to bytecode chunks without building an intermediate syntax tree (AST).
* **Precise Mark-and-Sweep Garbage Collector:** A custom-built tracing garbage collector that cleans up unused objects, handles cyclical references, and optimizes heap allocation.
* **Custom Hash Table:** Built from scratch to handle string interning, global/local variable bindings, and class method lookups.
* **Advanced Language Features:**
  * **Dynamic Typing:** Native support for numbers, booleans, strings, and nil.
  * **Lexical Closures:** Upvalues mapping to allow nested functions to access outer variables.
  * **Object-Oriented Programming:** Dynamic classes, instances, methods, fields, and inheritance.

---

## 🛠️ Project Structure

The project has been cleaned up and consolidated into a standard C workspace at the root level:

* `main.c` — Entry point, REPL loop, and script execution runner.
* `vm.c` / `vm.h` — The core virtual machine, execution loop, and stack runner.
* `compiler.c` / `compiler.h` — Single-pass compiler and Pratt parsing logic.
* `scanner.c` / `scanner.h` — Lexical scanner that produces tokens on demand.
* `object.c` / `object.h` — Representation of heap-allocated objects (strings, closures, classes).
* `table.c` / `table.h` — Custom hash table implementation.
* `memory.c` / `memory.h` — Core allocator and mark-and-sweep garbage collector.
* `chunk.c` / `chunk.h` — Bytecode representation and instruction arrays.
* `value.c` / `value.h` — Lox value representation and dynamic typing helper functions.
* `debug.c` / `debug.h` — Disassembler for inspecting generated bytecode instructions.

## ⚙️ How to Build and Run

### Prerequisites
Make sure you have a C compiler (`gcc` or `clang`) and `make` installed.

### Compilation
To compile the release build of the interpreter:
```bash
make
```

To compile a debug build (which includes compiler tracing and garbage collection logs):
```bash
make debug
```

### Running the Interpreter

**Interactive Mode (REPL):**
```bash
./clox
```

**Running a Lox script:**
```bash
./clox path/to/script.lox
```

### Cleaning Build Files
To remove object files and binaries:
```bash
make clean
```
